# CatchIt – Teil 2C
## Plan / Go / Recovery und Flowlogik

## 1. Plan Mode

Planung ist in CatchIt nicht Route Search, sondern Kettenaufbau.

### Muss leisten
- Vorhaben statt Provider-Parameter erfassen
- Folgeziele und Rückweg mitdenken
- Bewegungsstil und Tageslage abbilden
- alltagsnahe Fragen stellen

### Darf nicht werden
- Formularstrecke
- Settings-Dschungel
- technischer Filterdialog

---

## 2. Go Mode

Go Mode ist die Phase, in der nur das Nächste zählt.

### Hauptprinzip
**Exekution ist vertikal.**

Das heißt:
- Schrittfolge
- Handlung
- Zeitfluss
- Beruhigung und Fortschritt

### Kernzustände im Go Mode
- Countdown-geführter Main State
- Map Reveal
- Map Focus
- Walk to Stop
- In Vehicle

### Produktregel
Go Mode darf nie wie allgemeine Navigation wirken.  
Er bleibt CatchIt: ruhig, zustandsbasiert, handlungsfokussiert.

---

## 3. Recovery Mode

Recovery ist eine eigenständige Produkterfahrung.

### Recovery-Typen
- Simple Switch
- Recheck nötig
- Kette angepasst

### Recovery muss leisten
- einfachste Rettung zeigen
- neue Tragfähigkeit lesbar machen
- Last nicht unnötig an den Nutzer zurückgeben

### Recovery darf nicht werden
- Error Center
- Alternativenflut
- Vollvergleich
- technische Fehlermeldung

---

## 4. Map Reveal und Map Focus

### Main → Map Reveal
Map Reveal ist kein neuer Screen, sondern ein verdichteter Folgezustand.

Muss leisten:
- gleiche Travel Truth
- gleiche Systemfamilie
- Karte erscheint aus dem unteren Bereich
- Countdown bleibt lesbar
- keine Bottom-Sheet-Drift

### Map Focus
Map Focus macht Orientierung wichtiger, ohne den Zeitanker komplett zu verlieren.

Muss leisten:
- Raum lesen
- Zeit behalten
- keine Standard-Navi-Ästhetik

---

## 5. Flow-Regel

Planung, Exekution und Recovery dürfen **nicht** in derselben Oberfläche gleichberechtigt konkurrieren.

### Die App muss lesbar machen:
- Bin ich gerade im Vergleich?
- Bin ich in der Handlung?
- Reagiert das System?
- Muss ich selbst eingreifen?
