# CatchIt – Teil 3B
## Core Screens und Mockup-Logik

## 1. Main / Idle / Blue

### Zweck
Der Standard-Home-State.  
CatchIt erscheint als Zustandsoberfläche, nicht als Suchmaschine.

### Produktbotschaft
„Noch ist es ruhig. Alles ist vorbereitet.“

### Muss sichtbar sein
- ruhiger blauer State-Backdrop
- Countdown Anchor
- drei tragfähige Rollenoptionen
- kleiner Kettenkontext
- keine Karten-Dominanz

---

## 2. Main / Ready / Green

### Zweck
Der Übergang zur operativen Relevanz.

### Produktbotschaft
„Jetzt wird es relevant, aber du bist gut aufgestellt.“

### Muss sichtbar sein
- gleiche Struktur wie Blue
- stärkere Relevanz
- keine Alarmwirkung
- keine neue Produktgrammatik

---

## 3. Main / Adapting / Yellow

### Zweck
Änderung erkannt, CatchIt reagiert bereits.

### Produktbotschaft
„Etwas hat sich verändert. Ich passe das an.“

### Muss sichtbar sein
- Countdown bleibt primär
- Änderung ist lesbar
- leichte Reibung
- keine Warnästhetik
- keine Orange-/Red-Semantik

---

## 4. Main / Act Now / Orange

### Zweck
Der Nutzer muss jetzt handeln.

### Produktbotschaft
„Jetzt losgehen / jetzt handeln, dann bleibt es machbar.“

### Muss sichtbar sein
- Instruction Anchor
- hohe Exekutionslesbarkeit
- reduzierte Nebensignale
- Countdown nur noch trace-persistent

### Nicht tun
- CTA-Look
- Dashboard-Masse
- roter Krisenstil

---

## 5. Main / Failed / Red

### Zweck
Diese konkrete Lösung ist nicht mehr tragfähig.

### Produktbotschaft
„Das klappt so nicht mehr.“

### Muss sichtbar sein
- Failure Anchor
- selten, knapp, eindeutig
- kein Theater
- keine Datenflut

---

## 6. Main → Map Reveal

### Zweck
Räumliche Unterstützung wird freigelegt.

### Produktbotschaft
„Hier ist zusätzliche Raumhilfe, ohne dass du die Hauptwahrheit verlierst.“

### Muss sichtbar sein
- gleiche Oberfläche
- Karte wächst aus dem unteren Bereich
- Countdown bleibt lesbar
- kein Bottom-Sheet-Look

---

## 7. Map Focus

### Zweck
Orientierung wird wichtiger, aber Zeit bleibt präsent.

### Produktbotschaft
„Die Orientierung zählt jetzt stärker, aber der Takt bleibt maßgeblich.“

### Muss sichtbar sein
- größere Kartenfläche
- Bubble / Zeitanker weiter präsent
- keine Standard-Navi-Ästhetik
- operative Info untergeordnet, aber lesbar

---

## 8. Go Mode / Active Step

### Zweck
Nur das Nächste zählt.

### Produktbotschaft
„Das hier ist jetzt dran.“

### Muss sichtbar sein
- aktiver Schritt
- nächster direkter Folgeschritt
- zentrale Zeit-/Handlungsrelation
- vertikale Logik
- kaum Vergleich

---

## 9. Recovery / einfachste Rettung

### Zweck
Ein Plan kippt, CatchIt priorisiert die einfachste tragfähige Fortsetzung.

### Produktbotschaft
„Etwas ist nicht mehr stabil. Hier ist dein einfachster sicherer Weg weiter.“

### Muss sichtbar sein
- klarer Zustandswechsel
- eine beste Rettung
- optional ruhige Reserve
- Aussage zur Kette
