# CatchIt Design – Teil 1 (bereinigt)

Dieses ZIP ist die bereinigte, gesplittete Fassung von **CatchIt_Design_Teil_1.md**.

## Warum neu aufgesetzt
Die Ursprungsdatei enthält mehrere ältere Dokumentstränge mit inkonsistenter Terminologie und veraltetem Systemstand, unter anderem:
- alte 4-State-Logik
- uneinheitliche Main-State-Namen
- Vorstände vor Anchor-/Persistence-/Inheritance-Logik
- vermischte Konzept-, Screen- und Repo-Ebenen

## Ziel dieser Fassung
Teil 1 ist jetzt auf den **aktuellen CatchIt-Systemstand** gezogen:
- 5-State-Logik: Blue / Green / Yellow / Orange / Red
- Ketten statt Einzelfahrten
- Trust-/ND-/Privacy-Logik als Kern, nicht Zusatz
- CatchIt als zustandsbasiertes Mobility-Relief-System
- semantisch konsistente Begriffe
- schlankere, wartbare Split-Dateien

## Inhalt
1. `01_Produktkern_und_Differenzierung.md`
2. `02_Zustandslogik_und_UX_Grundprinzipien.md`
3. `03_Trust_Privacy_ND_Cathy.md`
4. `04_V1_Fokus_und_Repo_Alignment.md`

## Bearbeitungsprinzip
- keine Altlasten konservieren
- keine Mischterminologie
- keine alte 4-State-Nomenklatur
- keine klassischen Transit-App-Muster als Default
