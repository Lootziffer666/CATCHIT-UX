# Screen_06_Gold_Home_and_Family_Feste_Daten

## Purpose
Defines the gold-status version of the CatchIt Home & Family / feste Daten screen.

## Status
Gold-status supporting screen for CatchIt Layer 2.

## 1. CatchIt interpretation

This is not a profile page, contacts app, or family-management app.

It is the calm backstage surface for durable everyday anchors such as homes, children, and relevant people.

The point is not showing all personal data.
The point is storing stable life structure CatchIt may rely on.

## 2. State type

**System Layer / Backstage / Home & Family**

- Screen role: manage fixed places and relevant people
- Primary anchor: **Selection Anchor**
- User role: review and occasionally adjust durable structure
- CatchIt role: reflect lived family and location reality calmly
- Density: **low-to-medium**
- Tone: calm, clear, respectful, structurally useful

## 3. Must stay the same

- clearly backstage
- no giant form on first view
- no profile/social-app aesthetics
- no contacts-list behavior
- no map requirement
- places and people feel like daily anchors, not raw records
- sensitive information stays understated

## 4. Must change

Compared to Backstage Home, this becomes:
- concrete family/location structure
- still not full edit-form logic

First level should show only:
- **Zuhause**
- **Bezugspersonen**
- **Kinder / Wegebezug**
- optional **wichtige Orte**

Not everything fully expanded at once.

## 5. Drift risks

- profile drift
- form-wall drift
- contacts-app drift
- too much intimate information on first view
- homes as raw address list
- people as technical records

## 6. Recommended screen structure

### Header
Quiet, not profile-like.

Good title:
- **Zuhause & Familie**

### Orientation line
Optional:
- *Selten ändern, aber wichtig*

### Area 1 — Zuhause
Examples:
- **Zuhause 1** — *aktiv hinterlegt*
- **Zuhause 2** — *optional hinterlegt*

### Area 2 — Bezugspersonen / Kinder
Examples:
- **Luke** — *Kita verknüpft*
- **Jake** — *Schulweg relevant*

### Layout
- calm rows or grouped surfaces
- titles plus one small structural residue
- detail/edit only after tap

## 7. Prompt for generation tool

Design the CatchIt **Home & Family / feste Daten** screen as a calm backstage surface for durable personal structure such as homes, important people, and fixed everyday references, not as a profile page, contacts app, or large form screen.

Semantic meaning:
these are rarely changed foundations that allow CatchIt to understand recurring everyday mobility reality.

Requirements:
- keep the screen clearly in the Backstage/System Layer
- avoid profile-page aesthetics
- avoid contacts-app behavior
- avoid a giant editable form on first view
- show only a few clear structural groups
- keep sensitive information understated
- use quiet rows or soft grouped surfaces
