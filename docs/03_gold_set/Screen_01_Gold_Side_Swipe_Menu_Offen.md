# Screen_01_Gold_Side_Swipe_Menu_Offen

## Purpose
Defines the gold-status version of the CatchIt Side Swipe Menu / Open screen.

## Status
Gold-status supporting screen for CatchIt Layer 2.

## 1. CatchIt interpretation

This is not a classic navigation drawer and not a settings menu.

It is a contextual quick-adjust layer above the current CatchIt truth.

The active main state remains perceptible in the background as atmospheric residue.
The menu does not replace the current state.
It temporarily overlays it for fast today-relevant access.

## 2. State type

**Shell / Side Swipe Layer / Menu Open**

- Screen role: contextual quick-adjust and recall access
- Primary anchor: **Selection Anchor**-leaning
- User role: quickly adjust today or recall known routines
- CatchIt role: expose fast situational controls without creating admin feeling
- Density: **low-to-medium**
- Tone: calm, secondary, clear, non-administrative

## 3. Must stay the same

- the active main state remains perceptible in the background
- state color remains visible as atmospheric residue
- the menu feels like a layer, not a new app area
- entry count stays low
- grouping stays clear
- no dashboard behavior
- no standard drawer look
- no settings-app substitution
- no deep hierarchy on the first layer

## 4. Must change

When the layer opens:
- interaction focus moves from main state to side layer
- background becomes slightly blurred and dimmed
- background remains visible but not readable
- menu content is restricted to fast today-relevant controls and recall paths

Allowed groups:
- **Gespeicherte Pläne**
- **Heute anders**
- **Schnellzugriffe**

Allowed examples:
- Gespeicherte Pläne
- Heute mit Kind
- Heute wenig Fußweg
- Lieber laufen statt warten
- Heute einfacher

Not here:
- Zuhause 1 / 2
- Notfallkontakt-Verwaltung
- Family-Setup
- deep Vibration/Sound configuration

## 5. Drift risks

- classic drawer drift
- settings-panel drift
- too many entries
- too many submenu levels
- dashboard grouping
- glass-showcase effect
- fully erased background state
- admin wording instead of situational modulation

## 6. Recommended screen structure

### Background
- active Main screen remains visible
- moderate blur
- light dimming
- state color remains perceptible
- bubble and large forms remain as silhouette
- text is no longer readable

### Side layer surface
- one calm surface
- side-revealed, not many floating cards
- semitransparent or matte, but restrained
- clearly secondary to the main truth

### Header
- minimal or absent
- no heavy “Menu” heading
- optional small label such as:
  - **Heute**
  - **Schnellzugriff**

### Group 1 — Gespeicherte Pläne
2–4 entries only.

Examples:
- Kita → Ergo → Rückweg
- Schule morgens
- Luke abholen
- Standard-Arbeitstag

### Group 2 — Heute anders
Small, situational modifiers only.

Examples:
- Mit Kind
- Wenig Fußweg
- Lieber laufen
- Heute einfacher

### Group 3 — Schnellzugriffe
Few direct accesses only.

Examples:
- Gespeicherte Pläne öffnen
- Plan für heute prüfen
- Backstage öffnen

### Footer
No heavy footer.
At most one subordinate access:
- **Mehr Einstellungen**

## 7. Prompt for generation tool

Design the CatchIt **Side Swipe Menu / Open** screen as a contextual quick-adjust layer over the active main screen, not as a standard app drawer or settings menu.

Semantic meaning:
the current CatchIt state remains valid in the background, while the user opens a secondary layer for fast today-relevant adjustments and saved-plan recall.

Requirements:
- keep the active main screen visible in the background as blurred state residue
- preserve the background state color so the current CatchIt state is still perceptible
- make the background visible but not readable
- use one calm side-layer surface, not many floating cards
- keep the menu focused on fast, situational items only
- keep density low-to-medium
- keep the layer clearly secondary

Include groups like:
- Gespeicherte Pläne
- Heute anders
- Schnellzugriffe

Forbidden drifts:
- classic navigation drawer
- admin/settings panel
- dashboard grouping
- too many entries
- heavy glassmorphism showcase
- blacked-out background
- full loss of background state color
- deep multi-level menu behavior
