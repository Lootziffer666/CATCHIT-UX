# Screen_04_Gold_Saved_Plan_Quick_Change

## Purpose
Defines the gold-status version of the CatchIt Saved Plan Quick Change screen.

## Status
Gold-status supporting screen for CatchIt Layer 2.

## 1. CatchIt interpretation

This is not an editor, settings form, or full plan rebuild.

It is the calm local-adjustment surface for an already trusted chain.

The point is not reconfiguration.
The point is allowing one small change without losing continuity.

## 2. State type

**Shell / Recall Layer / Saved Plan Quick Change**

- Screen role: allow a small change to a saved plan
- Primary anchor: **Question Anchor** or calm **Confirmation Anchor**
- User role: adjust locally, not rebuild globally
- CatchIt role: preserve the chain while allowing today-specific deviation
- Density: **low-to-medium**
- Tone: calm, flexible, continuity-preserving

## 3. Must stay the same

- the saved plan remains clearly visible as the base
- the chain remains the main truth
- the change remains local
- no planner/editor aesthetics
- no form wall
- no technical routing language
- no route-results behavior

## 4. Must change

Compared to Quick Recall, this becomes:
- less pure confirmation
- more controlled deviation

Good quick-change cases:
- later bus
- short stop
- with child
- less walking today
- simpler today
- slower today

Not for:
- whole new day structure
- new homes/family data
- deep preference logic
- everything with many follow-up questions

## 5. Drift risks

- plan-editor drift
- settings/form drift
- too many change options
- large before/after views
- technical parameter logic
- toolbar overload
- backstage behavior instead of quick change

## 6. Recommended screen structure

### Header
Near the saved plan.

Example:
- **Kita → Ergo → Rückweg**
- *heute leicht anders*

### Primary anchor
Small clear adjustment framing.

Examples:
- **Was ist heute anders?**
- **Diesen Ablauf leicht anpassen**

### Base chain
The saved chain remains visible and compact.

Example:
- **Zuhause → Kita → Ergo → Kita → Zuhause**

### Quick-change choices
Few, clear, situational options.

Examples:
- **Späterer Bus**
- **Mit Kind**
- **Wenig Fußweg**
- **Kurzer Zwischenstopp**
- **Heute einfacher**

### Continuity residue
One line only.

Examples:
- *Rückweg bleibt mitgedacht*
- *Nur lokal angepasst*

### Primary action
Examples:
- **So übernehmen**
- **Für heute anpassen**

## 7. Prompt for generation tool

Design the CatchIt **Saved Plan Quick Change** screen as a calm local-adjustment surface for an already trusted recurring chain, not as a planner editor, settings form, or full plan rebuild screen.

Semantic meaning:
the saved plan remains the base, but one small today-specific change is needed without losing the chain.

Requirements:
- keep the saved chain clearly visible as the base truth
- frame the change as local and controlled
- keep density low-to-medium
- include only a small set of clearly situational quick-change options
- keep edit complexity low
- avoid form aesthetics
- avoid planner-editor behavior
- avoid route comparison
- avoid giant before/after differences
