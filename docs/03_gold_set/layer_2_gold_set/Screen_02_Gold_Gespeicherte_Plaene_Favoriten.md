# Screen_02_Gold_Gespeicherte_Plaene_Favoriten

## Purpose
Defines the gold-status version of the CatchIt Saved Plans / Favorites recall screen.

## Status
Gold-status supporting screen for CatchIt Layer 2.

## 1. CatchIt interpretation

This is not a bookmarks list, route history, or planner archive.

It is a calm recall surface for recurring, workable everyday chains.

The point is not data retrieval.
The point is re-entering trusted routines quickly.

## 2. State type

**Shell / Recall Layer / Saved Plans**

- Screen role: recall recurring plans quickly
- Primary anchor: **Selection Anchor** with calm recall emphasis
- User role: recognize and choose a known chain
- CatchIt role: make repeatable routines recognizable and reusable
- Density: **low-to-medium**
- Tone: calm, familiar, ordered, non-archival

## 3. Must stay the same

- clearly remains inside CatchIt
- no file/archive aesthetics
- no provider/history logic
- no route-results behavior
- no dashboard-card overload
- each entry must feel like a workable chain, not a data record
- a few clear groups, not an endless list
- recognition matters more than information completeness

## 4. Must change

Compared to the Side Swipe layer, this becomes:
- less toggle-like
- more routine-oriented
- more semantically clear per entry

Recommended groups:
- **Heute relevant**
- **Wiederkehrend**
- **Familie / Kinder**
- optional **Zuletzt genutzt**

Each item should show only:
- plan name
- compact chain
- one small routine/trust residue

Not on first view:
- full times
- full leg breakdown
- many badges
- technical metadata

## 5. Drift risks

- bookmarks-list drift
- route-history drift
- archive drift
- too many equal-weight cards
- too much metadata
- too many times on first level
- “my routes” look instead of routine look
- search/filter/sort overkill
- default map preview

## 6. Recommended screen structure

### Header
Quiet, small, not tool-like.

Good titles:
- **Gespeicherte Pläne**
- **Wiederkehrende Wege**
- **Deine Routinen**

### Group 1 — Heute relevant
2–3 likely important entries.

Examples:
- Kita → Ergo → Rückweg
- Schule morgens
- Luke abholen

### Group 2 — Wiederkehrend
Stable routine chains.

Examples:
- Standard-Arbeitstag
- Schule nachmittags
- Einkauf + Rückweg

### Item structure
Each item should show:
- primary plan name
- compact chain
- one small routine residue

Example:
- **Kita → Ergo → Rückweg**
- *mit Luke · Rückweg mitgedacht*

### Interaction
Tap leads to:
- **Saved Chain Detail / Quick Recall**

Optional subordinate action:
- **Jetzt nutzen**

### Layout
- calm
- lightly structured
- more ordered than Main
- no backoffice feeling
- no mini-card swarm
- no table aesthetics

## 7. Prompt for generation tool

Design the CatchIt **Gespeicherte Pläne / Favoriten** screen as a calm recall surface for recurring, workable everyday chains, not as a bookmarks list, route history, or planner archive.

Semantic meaning:
these are trusted, reusable routines that can be quickly reactivated.

Requirements:
- organize plans into a few calm groups
- make each saved entry feel like a recurring chain, not a saved search result
- show only what is needed for quick recognition
- keep density low-to-medium
- avoid map previews
- avoid heavy metadata
- avoid dashboard cards
- avoid route-results logic
- avoid archive/file-manager aesthetics
