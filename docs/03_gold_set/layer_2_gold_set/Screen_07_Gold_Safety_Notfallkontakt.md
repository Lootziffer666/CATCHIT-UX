# Screen_07_Gold_Safety_Notfallkontakt

## Purpose
Defines the gold-status version of the CatchIt Safety / Notfallkontakt screen.

## Status
Gold-status supporting screen for CatchIt Layer 2.

## 1. CatchIt interpretation

This is not an alarm center, panic UI, medical form system, or separate SOS app.

It is the calm backstage surface for a rare-but-important safety anchor.

The point is not dramatizing danger.
The point is storing one dependable emergency contact clearly and respectfully.

## 2. State type

**System Layer / Backstage / Safety / Notfallkontakt**

- Screen role: store and review emergency-contact basics
- Primary anchor: **Confirmation Anchor**-leaning
- User role: review, add, or rarely change a safety anchor
- CatchIt role: prepare reliable help without panic aesthetics
- Density: **low-to-medium**
- Tone: calm, clear, respectful, low-drama

## 3. Must stay the same

- clearly backstage
- no red warning world
- no crisis-center aesthetics
- no giant forms on first view
- no panic rhetoric
- safety remains clear but dignified
- contact feels like a dependable anchor, not a data wall

## 4. Must change

Compared to Home & Family, this becomes:
- one concrete safety foundation

First level should show only:
- whether a contact exists
- who it is
- one small reachability/role note
- one small trust/context note if useful

Not first:
- many emergency features
- many legal/privacy panels
- many editable fields

## 5. Drift risks

- alarm-center drift
- red crisis UI
- medical-form aesthetics
- too many fields
- safety feature-list behavior
- contacts-app drift
- unnecessary intimate exposure

## 6. Recommended screen structure

### Header
Quiet, non-dramatic.

Good title:
- **Notfallkontakt**

### Orientation line
Optional:
- *Selten gebraucht, aber wichtig*

### Main area
Example:
- **Christiane Amine**
- *mobil erreichbar*
- *im Ernstfall benachrichtigen*

If none exists:
- **Kein Notfallkontakt hinterlegt**
- *jetzt hinzufügen*

### Context line
Optional:
- *wird nicht im Alltag angezeigt*

### Actions
Primary:
- **Kontakt hinterlegen**
- **Kontakt ändern**

Secondary, reduced:
- **Entfernen**

## 7. Prompt for generation tool

Design the CatchIt **Safety / Notfallkontakt** screen as a calm backstage surface for storing and reviewing a rare-but-important emergency contact, not as an alarm center, panic UI, medical form system, or separate SOS app.

Semantic meaning:
this is a quiet safety foundation that CatchIt can rely on if needed.

Requirements:
- keep the screen clearly in the Backstage/System Layer
- avoid red warning-world styling
- avoid crisis-center aesthetics
- avoid giant form walls on first view
- show only the core emergency-contact structure
- keep actions very limited
- keep the tone respectful and low-drama
