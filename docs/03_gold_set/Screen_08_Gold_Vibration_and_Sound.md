# Screen_08_Gold_Vibration_and_Sound

## Purpose
Defines the gold-status version of the CatchIt Vibration & Sound overview screen.

## Status
Gold-status supporting screen for CatchIt Layer 2.

## 1. CatchIt interpretation

This is not an audio panel, haptics lab, or technical settings matrix.

It is the calm backstage surface for how CatchIt may support the user through sound and haptics.

The point is not exposing every parameter.
The point is making sensory support understandable, humane, and situationally useful.

## 2. State type

**System Layer / Backstage / Vibration & Sound**

- Screen role: structure sensory feedback settings
- Primary anchor: **Selection Anchor**
- User role: decide how CatchIt may feel or sound
- CatchIt role: support states through calm sensory logic
- Density: **low-to-medium**
- Tone: calm, functional, understandable, non-technical

## 3. Must stay the same

- clearly backstage
- no developer-settings feel
- no giant switch/slider matrix
- no sterile settings grid
- no sound/haptics showcase
- focus remains daily support, not effect control
- a few clear grouped areas only

## 4. Must change

Compared to Safety, this becomes:
- sensory system overview

The first level should show only:
- **Vibration**
- **Sound**
- optional **Modus**

It should answer:
- does CatchIt signal physically?
- does CatchIt signal audibly?
- how calm or strong is it?

Not first:
- pattern editors
- long trigger lists
- waveform jargon
- hardware-centric controls

## 5. Drift risks

- audio-panel drift
- haptics-lab drift
- too many controls at once
- too many trigger categories
- tech/nerd settings tone
- effect selection instead of daily support

## 6. Recommended screen structure

### Header
Quiet, not technocratic.

Good title:
- **Vibration & Sound**

### Orientation line
Optional:
- *So meldet CatchIt sich bei dir*

### Area 1 — Vibration
Examples:
- **Vibration** — *ruhig aktiviert*
- **Vibration** — *deutlich*

### Area 2 — Sound
Examples:
- **Sound** — *dezent*
- **Sound** — *nur wichtige Hinweise*

### Area 3 — Modus
Examples:
- **Modus** — *ruhig*
- **Modus** — *nur kritisch*

### Actions
Tap opens smaller sub-surfaces.
Optional secondary action:
- **Testen**
- **Anpassen**

## 7. Prompt for generation tool

Design the CatchIt **Vibration & Sound** screen as a calm backstage surface for configuring sensory feedback, not as an audio control panel, haptics lab, or technical settings matrix.

Semantic meaning:
this is where CatchIt’s sensory support is structured so that states can be felt or heard in an understandable, low-stress way.

Requirements:
- keep the screen clearly in the Backstage/System Layer
- avoid technical overload
- avoid giant lists of switches and sliders on first view
- avoid audio-tool or developer-settings aesthetics
- show only a few high-level sensory areas such as:
  - Vibration
  - Sound
  - optional Modus
- keep actions limited and secondary
