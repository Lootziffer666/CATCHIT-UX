# Inventory and Classification (Audit Snapshot)

## Canonical-intent documents identified
- Philosophy framing docs (formerly in `new/`): CatchIt philosophy, design guidelines, design OS, startup rules.
- System-rule docs: state, anchor, transitions/persistence, density/spatial/motion, copy tone, additional rules, implementation bridge.
- Gold references: base/core/extended/masterlist and visual baseline.

## Supporting implementation/spec documents identified
- Figma screen specs (parts 1-5).
- Figma property/token specs (parts 1-4).
- Compose Android translation docs (parts 1-5).
- Implementation-near specs and code scaffold notes.
- Split design parts (`CatchIt_Design_Teil_*_bereinigt_SPLIT`).

## Prototype/archive artifacts identified
- HTML single-file prototypes and onboarding mocks.
- ZIP/RAR packages for prototypes, design packs, and historical exports.

## Legacy/deprecated area identified
- `maybe_deprecated/` contains archive analyses, deprecated prompts, and prior generated outputs.

## Duplicate/overlap signals
- Multiple packaging exports for similar content (`catchit_prototype*`, `catchit_screen_spec_pack*`, `Catchit_design_pack_*`).
- Parallel document tracks (`CatchIt_Design_Dokument_*`, `CatchIt_Design_Teil_*`, split/unsplit variants).

These are preserved but separated by authority to avoid accidental canonical drift.
