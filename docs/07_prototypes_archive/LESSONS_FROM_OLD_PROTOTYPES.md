# Lessons from Old Prototypes

## What old prototypes did well
- Often sharper anchor readability and immediate orientation.
- Clearer separation between planning and execution moments.
- Strong reduction in on-screen noise for day/chain readability.
- Fast comprehension through direct interaction clarity.

## What should still inform current work
- Harder anchor readability should remain a design check.
- Planning should stay horizontal; execution should stay vertical.
- Reduction should protect confidence, not remove necessary control.
- Day/chain readability should remain simple and immediate.
- Prototype clarity should actively inform current refinements.

## What must not be copied blindly
- Any prototype shortcut that breaks Main/Side Swipe/Backstage boundaries.
- Legacy patterns that bypass current canonical state/anchor logic.
- Visual experiments treated as rules without canonical/gold promotion.

Old prototypes are reference material: neither romanticized nor dismissed.
